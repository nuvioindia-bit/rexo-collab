# आवश्यकता दस्तावेज़

## 1. एप्लिकेशन अवलोकन

**एप्लिकेशन नाम**: Rexo Collab

**विवरण**: एक मोबाइल-फर्स्ट क्रिएटर मार्केटप्लेस PWA जो ब्रांड्स को क्रिएटर्स/इन्फ्लुएंसर्स से जोड़ता है। ब्रांड्स कैंपेन पोस्ट करते हैं, क्रिएटर्स ब्राउज़ करके अप्लाई करते हैं, और प्लेटफॉर्म एप्लिकेशन मैनेजमेंट और पेमेंट प्रोसेसिंग के माध्यम से सहयोग की सुविधा देता है। पहला लॉगिन करने वाला यूज़र Super Admin बनता है जो पूरे एप्लिकेशन को कंट्रोल कर सकता है।

---

## 2. उपयोगकर्ता और उपयोग परिदृश्य

### 2.1 लक्षित उपयोगकर्ता

**ब्रांड यूज़र्स**:
- क्रिएटर पार्टनरशिप चाहने वाली कंपनियां
- इन्फ्लुएंसर कैंपेन मैनेज करने वाली मार्केटिंग टीमें

**क्रिएटर यूज़र्स**:
- कंटेंट क्रिएटर्स और इन्फ्लुएंसर्स
- ब्रांड कोलैबोरेशन चाहने वाले सोशल मीडिया पर्सनैलिटीज़

**Super Admin यूज़र**:
- पहला रजिस्टर और लॉगिन करने वाला यूज़र
- पूरे एप्लिकेशन का कंट्रोल

### 2.2 मुख्य उपयोग परिदृश्य

**ब्रांड परिदृश्य**:
- आवश्यकताओं और बजट के साथ कैंपेन पोस्ट करना
- क्रिएटर एप्लिकेशन्स रिव्यू करना
- कोलैबोरेशन सेलेक्ट और मैनेज करना
- वॉलेट के माध्यम से पेमेंट प्रोसेस करना

**क्रिएटर परिदृश्य**:
- उपलब्ध कैंपेन ब्राउज़ करना
- प्रासंगिक अवसरों के लिए अप्लाई करना
- प्रोफाइल और पोर्टफोलियो मैनेज करना
- वॉलेट के माध्यम से पेमेंट प्राप्त करना

**Super Admin परिदृश्य**:
- बैनर/पोस्टर मैनेजमेंट
- यूज़र मैनेजमेंट
- पेमेंट मैनेजमेंट
- सभी कंटेंट को एडिट/रिमूव करना

## 3. पेज संरचना और कार्यक्षमता

### 3.1 पेज पदानुक्रम

```
एप्लिकेशन रूट
├── Authentication
│   ├── Login
│   └── Registration (Role Selection: Brand/Creator)
├── Main Navigation (Bottom Bar)
│   ├── Home
│   ├── Campaigns
│   ├── Wallet
│   └── Profile
├── Side Drawer Menu
│   ├── Account Settings
│   ├── Notifications Settings
│   ├── Security Settings
│   ├── Connected Socials
│   ├── Admin Panel (Super Admin Only)
│   └── Logout
├── Campaign Details
│   └── Application Flow
└── Admin Panel
    ├── Banner Management
    ├── User Management
    └── Payment Management
```

### 3.2 Authentication Pages

#### 3.2.1 Registration Page
- यूज़र टाइप सेलेक्शन (Brand / Creator)
- ईमेल इनपुट
- पासवर्ड इनपुट
- रजिस्ट्रेशन सबमिशन
- Firebase Authentication से यूज़र बनाना
- Firestore में यूज़र रोल सेव करना
- पहले यूज़र को Super Admin के रूप में मार्क करना
- सफल रजिस्ट्रेशन के बाद लॉगिन पर नेविगेट करना

#### 3.2.2 Login Page
- ईमेल इनपुट
- पासवर्ड इनपुट
- Firebase Authentication से लॉगिन
- Auth state persistence
- सफल लॉगिन के बाद होम पर नेविगेट करना

### 3.3 Home Screen

#### 3.3.1 Auto-Sliding Banner Carousel
- हेडर के ठीक नीचे फुल-विड्थ कैरोसेल
- बैनर/पोस्टर ऑटोमैटिक स्लाइड होते हैं (3-4 सेकंड इंटरवल)
- हर पोस्टर पर टाइटल और लिंक
- क्लिक करने पर रीडायरेक्ट
- टच/स्वाइप सपोर्ट
- Firestore से रियल-टाइम बैनर डेटा लोड होता है

#### 3.3.2 Creator Home Content
- फीचर्ड कैंपेन कैरोसेल
- पर्सनल स्टैट्स ओवरव्यू (सबमिट किए गए एप्लिकेशन, एक्टिव कोलैबोरेशन)
- ट्रेंडिंग कैंपेन लिस्ट
- कैंपेन के लिए अप्लाई करने का क्विक एक्सेस

#### 3.3.3 Brand Home Content
- एक्टिव कैंपेन ओवरव्यू
- एप्लिकेशन स्टैटिस्टिक्स (पेंडिंग, अप्रूव्ड, रिजेक्टेड)
- नया कैंपेन पोस्ट करने का क्विक एक्सेस
- हाल के क्रिएटर एप्लिकेशन

### 3.4 Campaigns Screen

#### 3.4.1 Creator View
- फिल्टर्स के साथ कैंपेन लिस्ट
- हर कैंपेन कार्ड में:
  - ब्रांड लोगो
  - कैंपेन टाइटल
  - पेआउट अमाउंट
  - क्रिएटर आवश्यकताएं
  - एप्लिकेशन डेडलाइन
  - अप्लाई बटन
- सर्च फंक्शनैलिटी
- कैटेगरी, पेआउट रेंज, डेडलाइन से फिल्टर

#### 3.4.2 Brand View
- अपने पोस्ट किए गए कैंपेन की लिस्ट
- कैंपेन स्टेटस (एक्टिव, क्लोज़्ड, ड्राफ्ट)
- नया कैंपेन बनाने का बटन
- कैंपेन एडिट/डिलीट ऑप्शन

### 3.5 Campaign Details Page

#### 3.5.1 Creator View
- पूरा कैंपेन विवरण
- ब्रांड जानकारी
- विस्तृत आवश्यकताएं
- अपेक्षित डिलीवरेबल्स
- टाइमलाइन और डेडलाइन
- पेआउट विवरण
- अप्लाई बटन (यदि पहले से अप्लाई नहीं किया)
- एप्लिकेशन स्टेटस (यदि पहले से अप्लाई किया)

#### 3.5.2 Brand View
- कैंपेन ओवरव्यू
- प्राप्त एप्लिकेशन की लिस्ट
- एप्लिकेशन रिव्यू इंटरफेस
- एप्लिकेशन अप्रूव/रिजेक्ट एक्शन

### 3.6 Creator Application Flow

कैंपेन डिटेल्स से एक्सेस किया जाने वाला मल्टी-स्टेप फॉर्म:

**स्टेप 1: कैटेगरी सेलेक्शन**
- कंटेंट कैटेगरी सेलेक्ट करें (Fashion, Tech, Lifestyle, आदि)

**स्टेप 2: ऑडियंस स्टैटिस्टिक्स**
- कुल फॉलोअर्स
- औसत एंगेजमेंट रेट
- प्राथमिक प्लेटफॉर्म

**स्टेप 3: सोशल लिंक्स**
- Instagram URL
- YouTube URL
- TikTok URL
- अन्य प्लेटफॉर्म

**स्टेप 4: प्राइसिंग**
- कैंपेन के लिए प्रस्तावित रेट
- नेगोशिएबल ऑप्शन

**स्टेप 5: पोर्टफोलियो**
- पिछले काम के सैंपल अपलोड करें
- पोर्टफोलियो लिंक जोड़ें

**स्टेप 6: सबमिट**
- सभी जानकारी रिव्यू करें
- एप्लिकेशन सबमिट करें
- कन्फर्मेशन मैसेज

### 3.7 Wallet Screen

#### 3.7.1 Creator View
- करंट बैलेंस डिस्प्ले
- कुल कमाई दिखाने वाला अर्निंग कार्ड
- पेआउट हिस्ट्री लिस्ट:
  - कैंपेन नाम
  - अमाउंट
  - तारीख
  - स्टेटस (पेंडिंग, कम्प्लीटेड)
- विड्रॉ बटन
- ट्रांजैक्शन डिटेल्स

#### 3.7.2 Brand View
- पेमेंट हिस्ट्री
- पेंडिंग पेमेंट्स
- कम्प्लीटेड ट्रांजैक्शन
- पेमेंट मेथड मैनेजमेंट

### 3.8 Profile Screen

#### 3.8.1 Creator Profile
- प्रोफाइल फोटो
- बायो

  - कुल एप्लिकेशन

- एडिट प्रोफाइल बटन
- सेटिंग्स एक्सेस

#### 3.8.2 Brand Profile
- कंपनी लोगो
- कंपनी विवरण
- कैंपेन स्टैटिस्टिक्स:
  - कुल पोस्ट किए गए कैंपेन

- एडिट प्रोफाइल बटन
- सेटिंग्स एक्सेस

### 3.9 Settings (Side Drawer Menu)

#### 3.9.1 Account Settings
- ईमेल एडिट करें
- पासवर्ड बदलें
- अकाउंट टाइप डिस्प्ले

#### 3.9.2 Notifications Settings
- ईमेल नोटिफिकेशन टॉगल
- पुश नोटिफिकेशन टॉगल
- टाइप के अनुसार नोटिफिकेशन प्रेफरेंस

#### 3.9.3 Security Settings
- टू-फैक्टर ऑथेंटिकेशन टॉगल
- लॉगिन हिस्ट्री
- एक्टिव सेशन मैनेजमेंट

#### 3.9.4 Connected Socials
- सोशल मीडिया अकाउंट लिंक/अनलिंक करें
- अकाउंट ओनरशिप वेरिफाई करें

#### 3.9.5 Admin Panel (Super Admin Only)
- बैनर मैनेजमेंट एक्सेस
- यूज़र मैनेजमेंट एक्सेस
- पेमेंट मैनेजमेंट एक्सेस

#### 3.9.6 Logout
- लॉगआउट कन्फर्मेशन
- सेशन क्लियर करें और लॉगिन पर रिटर्न करें

### 3.10 Admin Panel (Super Admin Only)

#### 3.10.1 Banner Management
- मौजूदा बैनर/पोस्टर की लिस्ट
- नया पोस्टर अपलोड करें:
  - इमेज अपलोड (Firebase Storage में)
  - पोस्टर टाइटल जोड़ें
  - पोस्टर लिंक (रीडायरेक्ट URL) जोड़ें
- मौजूदा पोस्टर डिलीट करें
- पोस्टर का ऑर्डर बदलें (drag या up/down बटन)
- Firestore में रियल-टाइम सेव

#### 3.10.2 User Management
- सभी यूज़र्स की लिस्ट
- यूज़र डिटेल्स देखें
- यूज़र एडिट करें
- यूज़र रिमूव करें

#### 3.10.3 Payment Management
- सभी ट्रांजैक्शन देखें
- पेमेंट एडिट करें
- पेमेंट रिमूव करें

### 3.11 Navigation Components

#### 3.11.1 Bottom Navigation Bar
- चार टैब: Home, Campaigns, Wallet, Profile
- एक्टिव टैब इंडिकेटर
- टैब स्विचिंग फंक्शनैलिटी
- कंटेंट के ऊपर ओवरलैप नहीं होना
- कंटेंट को padding-bottom मिलना

#### 3.11.2 Top Menu Button
- तीन-लाइन मेनू आइकन (टॉप-लेफ्ट)
- टैप करने पर साइड ड्रॉअर खुलता है

#### 3.11.3 Side Drawer
- ओवरले बैकग्राउंड
- मेनू आइटम:
  - Account
  - Notifications
  - Security
  - Connected Socials
  - Admin Panel (Super Admin के लिए)
  - Logout
- सेलेक्शन या बैकग्राउंड टैप पर ड्रॉअर बंद होना

## 4. व्यावसायिक नियम और तर्क

### 4.1 यूज़र रजिस्ट्रेशन और लॉगिन
- यूज़र्स को Brand या Creator के रूप में रजिस्टर करना होगा
- रजिस्ट्रेशन के बाद यूज़र टाइप नहीं बदला जा सकता
- ईमेल यूनीक होना चाहिए
- पासवर्ड न्यूनतम सुरक्षा आवश्यकताओं को पूरा करना चाहिए
- Firebase Authentication से यूज़र बनाना
- Firestore में यूज़र रोल स्टोर करना
- पहला रजिस्टर होने वाला यूज़र Super Admin बनता है

### 4.2 Super Admin अधिकार
- पहला लॉगिन करने वाला यूज़र Super Admin बनता है
- Super Admin सभी बैनर/पोस्टर मैनेज कर सकता है
- Super Admin सभी यूज़र्स को एडिट/रिमूव कर सकता है
- Super Admin सभी पेमेंट्स को मैनेज कर सकता है
- Super Admin एप्लिकेशन में उपलब्ध सभी कंटेंट को कंट्रोल कर सकता है

### 4.3 Banner/Poster Management
- बैनर Firestore में स्टोर होते हैं
- बैनर में इमेज URL, टाइटल, रीडायरेक्ट लिंक होता है
- बैनर का ऑर्डर बदला जा सकता है
- Home screen पर बैनर ऑटोमैटिक स्लाइड होते हैं
- बैनर पर क्लिक करने पर रीडायरेक्ट होता है

### 4.4 Campaign Posting (Brands Only)
- ब्रांड्स अनलिमिटेड कैंपेन पोस्ट कर सकते हैं
- हर कैंपेन में शामिल होना चाहिए:
  - टाइटल
  - विवरण
  - आवश्यकताएं
  - पेआउट अमाउंट
  - डेडलाइन
- कैंपेन को ड्राफ्ट के रूप में सेव किया जा सकता है
- एक्टिव कैंपेन सभी क्रिएटर्स को दिखाई देते हैं

### 4.5 Campaign Application (Creators Only)
- क्रिएटर्स कई कैंपेन के लिए अप्लाई कर सकते हैं
- प्रति कैंपेन प्रति क्रिएटर एक एप्लिकेशन
- सबमिशन के बाद एप्लिकेशन एडिट नहीं किया जा सकता
- एप्लिकेशन स्टेटस: Pending, Approved, Rejected

### 4.6 Application Review (Brands Only)
- ब्रांड्स अपने कैंपेन के लिए एप्लिकेशन रिव्यू करते हैं
- ब्रांड्स एप्लिकेशन अप्रूव या रिजेक्ट कर सकते हैं
- अप्रूव होने पर क्रिएटर को नोटिफाई किया जाता है
- रिजेक्टेड एप्लिकेशन दोबारा सबमिट नहीं किए जा सकते

### 4.7 Wallet और Payments
- क्रिएटर्स कैंपेन कम्प्लीशन के बाद पेमेंट प्राप्त करते हैं
- पेमेंट वॉलेट में प्रोसेस होते हैं
- क्रिएटर्स वॉलेट से फंड विड्रॉ कर सकते हैं
- न्यूनतम विड्रॉ अमाउंट लागू होता है
- ट्रांजैक्शन हिस्ट्री मेंटेन की जाती है

### 4.8 Profile Management
- यूज़र्स प्रोफाइल जानकारी एडिट कर सकते हैं
- क्रिएटर्स को अपडेटेड सोशल लिंक्स मेंटेन करने होंगे
- प्रोफाइल कम्प्लीटनेस विज़िबिलिटी को प्रभावित करती है

### 4.9 Mobile Layout और Responsive
- स्टेटस बार/नॉच के लिए safe-area-inset फिक्स
- टॉप में स्क्रीन कट नहीं होना, फुल स्क्रीन शो होना
- बॉटम नेव कंटेंट के ऊपर ओवरलैप नहीं होना
- हर मोबाइल स्क्रीन साइज़ (375px से 430px तक) पर ऑटोमैटिक फिट होना
- कोई हॉरिजॉन्टल स्क्रॉल नहीं, कोई क्लिपिंग नहीं

### 4.10 Firebase Integration
- Firebase Authentication यूज़र लॉगिन/रजिस्ट्रेशन के लिए
- Firestore डेटा स्टोरेज के लिए
- Firebase Storage इमेज अपलोड के लिए
- कोई dummy/mock डेटा नहीं, सभी डेटा Firebase से आता है

## 6. स्वीकृति मानदंड

**Super Admin सेटअप जर्नी**:
1. पहला यूज़र ईमेल और पासवर्ड से रजिस्टर करता है
2. सिस्टम उसे Super Admin के रूप में मार्क करता है
3. Super Admin लॉगिन करता है
4. Super Admin साइड ड्रॉअर में Admin Panel एक्सेस करता है
5. Super Admin Banner Management में जाता है
6. Super Admin नया बैनर अपलोड करता है (इमेज, टाइटल, लिंक)
7. बैनर Home screen पर दिखाई देता है और ऑटो-स्लाइड होता है
8. Super Admin बैनर का ऑर्डर बदलता है

**Creator User Journey**:
1. Creator ईमेल और पासवर्ड से रजिस्टर करता है
2. Creator सोशल लिंक्स और पोर्टफोलियो के साथ प्रोफाइल कम्प्लीट करता है
3. Creator Home screen पर ऑटो-स्लाइडिंग बैनर देखता है
4. Creator Campaigns screen पर कैंपेन ब्राउज़ करता है
5. Creator एक कैंपेन सेलेक्ट करके डिटेल्स देखता है
6. Creator मल्टी-स्टेप एप्लिकेशन फॉर्म कम्प्लीट करता है
7. Creator एप्लिकेशन सबमिट करता है और कन्फर्मेशन प्राप्त करता है
8. Creator Profile screen पर एप्लिकेशन स्टेटस चेक करता है
9. अप्रूवल के बाद, Creator Wallet में पेमेंट देखता है

**Brand User Journey**:
1. Brand ईमेल और पासवर्ड से रजिस्टर करता है
2. Brand कंपनी प्रोफाइल कम्प्लीट करता है
3. Brand आवश्यकताओं और बजट के साथ नया कैंपेन बनाता है
4. Brand कैंपेन पब्लिश करता है
5. Brand क्रिएटर एप्लिकेशन प्राप्त करता है
6. Brand एप्लिकेशन रिव्यू करता है और उपयुक्त क्रिएटर को अप्रूव करता है
7. Brand कैंपेन कम्प्लीशन के बाद Wallet के माध्यम से पेमेंट प्रोसेस करता है
